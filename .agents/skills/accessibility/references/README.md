# Accessibility References

## Purpose

This directory contains reference materials related to accessibility standards and best practices.

These documents are intended to support AI agents and developers when implementing accessible user interfaces.

The files in this directory are reference resources only.

They do not define project rules.

Project-specific accessibility requirements are documented in:

../SKILL.md

---

## Reference Categories

Reference documents may include:

- WCAG Guidelines
- WAI-ARIA Specifications
- HTML Accessibility
- Semantic HTML Examples
- Keyboard Navigation
- Screen Reader Support
- Color Contrast Guidelines
- Accessible Form Design
- Accessible SVG Examples
- Mobile Accessibility
- Accessibility Testing Guides

---

## Intended Use

AI agents should use these documents to:

- verify accessibility recommendations
- understand implementation details
- follow established accessibility standards
- avoid introducing accessibility regressions

---

Use descriptive file names.

wcag-2.2.md

## Guidelines

- Keep references technology-neutral whenever possible.
- Prefer official specifications.
- Avoid duplicate documentation.
- Update references when standards evolve.
- Remove obsolete guidance.
