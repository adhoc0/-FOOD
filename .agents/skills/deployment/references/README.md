# Deployment References

## Purpose

This directory contains reference documentation related to deployment strategies, best practices, CI/CD pipelines, and infrastructure management.

These documents are intended to support AI agents and developers when designing and maintaining deployment workflows.

Project-specific deployment rules are defined in:

../SKILL.md

---

## Reference Categories

Reference documents may include:

- CI/CD Best Practices
- Deployment Strategies (Blue/Green, Canary, Rolling)
- Infrastructure as Code (Terraform, Pulumi)
- Containerization (Docker, Kubernetes)
- Serverless Deployments
- Monitoring and Logging
- Security Best Practices
- Rollback Procedures
- Cloud Provider Specifics
- Performance Optimization

---

## Intended Use

AI agents should use these documents to:

- Understand deployment patterns
- Choose appropriate strategies
- Implement CI/CD pipelines
- Ensure secure deployments
- Optimize infrastructure
- Plan rollback procedures

---

## Guidelines

- Keep references technology-neutral where possible.
- Prefer official specifications.
- Avoid duplicate documentation.
- Update references when standards evolve.
- Remove obsolete guidance.
