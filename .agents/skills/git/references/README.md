# Git References

## Purpose

This directory contains reference documentation for Git and version control.

These documents provide implementation guidance, workflow examples, and best practices for developers and AI agents.

Project-specific Git rules are defined in:

../SKILL.md

---

## Scope

Reference documents may include:

- Git Basics
- Branching Strategies
- Semantic Versioning
- GitFlow
- GitHub Flow
- Commit Conventions
- Pull Requests
- Merge Strategies
- Conflict Resolution
- Repository Maintenance

---

## Intended Use

Reference documents should help:

- Explain Git concepts
- Document workflows
- Provide best practices
- Reduce duplicated knowledge

## File Naming

Use descriptive lowercase names.

Examples:

branching.md

gitflow.md

semantic-versioning.md

Avoid:

notes.md

temp.md

misc.md

---

## Guidelines

Reference documents should:

- Focus on one topic.
- Be concise.
- Be technology-specific.
- Avoid duplicate information.
- Reference official Git documentation whenever appropriate.
