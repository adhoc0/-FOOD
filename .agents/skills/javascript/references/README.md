# JavaScript References

## Purpose

This directory contains reference documentation for JavaScript development used in the FOOD project.

These documents provide implementation guidance, language features, browser APIs, coding patterns, and best practices for developers and AI agents.

Project-specific JavaScript rules are defined in:

../SKILL.md

---

## Scope

Reference documents may cover topics such as:

- JavaScript Language Features
- ES6+
- DOM Manipulation
- Browser APIs
- Event Handling
- Modules
- Async Programming
- Fetch API
- JSON
- Forms
- Validation
- Accessibility
- Performance
- Error Handling
- Browser Compatibility

---

## Intended Use

Reference documents should help:

- Explain JavaScript concepts.
- Provide implementation examples.
- Document best practices.
- Support AI-assisted development.
- Improve maintainability.
- Improve performance.
- Reduce duplicated knowledge.

## File Naming

Use descriptive lowercase filenames.

Examples:

```text
fetch-api.md
async-await.md
browser-support.md
error-handling.md
```

Avoid:

```text
notes.md
temp.md
misc.md
new.md
```

---

## Guidelines

Reference documents should:

- Focus on a single topic.
- Be JavaScript-specific.
- Be concise and practical.
- Avoid duplicate information.
- Reference official ECMAScript and MDN documentation whenever appropriate.
- Be reviewed periodically.
