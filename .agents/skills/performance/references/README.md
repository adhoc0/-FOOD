# Performance References

## Purpose

This directory contains reference documentation for application performance, scalability, optimization techniques, monitoring, and profiling used in the FOOD project.

These documents provide implementation guidance, performance analysis methods, benchmarking techniques, and best practices for developers and AI agents.

Project-specific performance rules are defined in:

../SKILL.md

---

## Scope

Reference documents may cover topics such as:

- Performance Fundamentals
- Performance Metrics
- Benchmarking
- Profiling
- Database Performance
- ORM Optimization
- Query Optimization
- Caching
- Memory Management
- CPU Optimization
- I/O Optimization
- Network Performance
- Frontend Performance
- Backend Performance
- Core Web Vitals
- Monitoring
- Logging
- Load Testing
- Stress Testing
- Scalability

---

## Intended Use

Reference documents should help:

- Explain performance concepts.
- Provide optimization techniques.
- Document benchmarking methods.
- Describe profiling strategies.
- Support AI-assisted performance improvements.
- Reduce duplicated knowledge.

## File Naming

Use descriptive lowercase filenames.

Examples:

```text
query-optimization.md
core-web-vitals.md
memory-management.md
```

Avoid:

```text
notes.md
temp.md
misc.md
new.md
```

---

## Guidelines

Reference documents should:

- Focus on a single topic.
- Be performance-specific.
- Be concise and practical.
- Avoid duplicate information.
- Reference official documentation whenever appropriate.
- Include measurable performance metrics where possible.
- Be reviewed periodically.
