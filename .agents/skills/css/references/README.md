# CSS References

## Purpose

This directory contains reference documentation related to CSS standards, methodologies, browser compatibility, responsive design, accessibility, and performance.

The documents in this directory provide implementation guidance and technical references for developers and AI agents.

Project-specific CSS rules are defined in:

../SKILL.md

---

## Intended Use

Reference documents may be used to:

- Understand CSS best practices
- Follow modern CSS standards
- Verify implementation details
- Improve maintainability
- Improve performance
- Improve accessibility
- Maintain design consistency

## File Naming

Use descriptive, lowercase file names.

Examples:

```text
responsive-design.md
css-grid.md
design-tokens.md
browser-support.md
```

Avoid:

```text
notes.md
temp.md
new.md
misc.md
```

---

## Guidelines

Reference documents should:

- Be technology-focused.
- Remain framework-independent whenever possible.
- Be concise and practical.
- Avoid duplicate information.
- Reference official specifications whenever available.
- Be reviewed periodically.
